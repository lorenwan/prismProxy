# 后端测试规范

## 测试策略

### 测试金字塔

| 层级 | 占比 | 说明 |
|------|------|------|
| 单元测试 | 70% | 快速、独立、可重复 |
| 集成测试 | 20% | 验证模块间交互 |
| E2E 测试 | 10% | 验证完整流程 |

### 测试原则

1. **快速** - 测试应该在秒级完成
2. **独立** - 测试之间不相互依赖
3. **可重复** - 任何环境都能运行
4. **自验证** - 自动判断通过/失败

## 测试分类

### 表驱动测试

- 使用结构体切片定义测试用例
- 每个用例包含：名称、输入、期望输出、是否报错
- 使用 `t.Run` 运行子测试

### Mock 策略

- 接口 Mock：实现接口的 mock 版本
- 依赖注入：通过构造函数注入 mock 对象
- 错误模拟：mock 返回错误以测试错误处理

### gRPC 服务测试

- 测试请求验证
- 测试响应格式
- 测试错误处理

---

## 测试覆盖率

| 类型 | 要求 | 说明 |
|------|------|------|
| 新代码 | ≥ 80% | 所有新功能必须有测试 |
| 现有代码 | 逐步提升 | 不强制一次性达标 |
| 关键路径 | 100% | 核心业务逻辑必须覆盖 |

---

## 测试命名规范

- 测试文件：`[模块名]_test.go`
- 测试函数：`Test[类型]_[方法]`
- 测试描述：使用中文，描述行为而非实现
- 测试结构：Arrange → Act → Assert
